| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |

## 1. Purpose

This policy establishes password requirements to protect company systems, data, and user accounts from unauthorized access.

## 2. Scope

### 2.1 Applicability

This policy applies to:

* All employees
* Contractors and consultants
* Third-party vendors with system access

### 2.2 Systems/Assets Covered

* Workstations
* Servers
* Cloud services
* Email accounts
* Business applications

### 2.3 Exclusions

* Personal systems not connected to company resources

## 3. Policy Statements

### 3.1 Password Requirements

All users must create strong passwords.

Requirements:

* Minimum 12 characters
* At least one uppercase letter
* At least one lowercase letter
* At least one number
* At least one special character

### 3.2 Password Protection

Users must protect their passwords.

Requirements:

* Password sharing is prohibited
* Passwords must not be written in visible locations
* Default passwords must be changed immediately

### 3.3 Account Security

Additional controls must be used.

Requirements:

* MFA must be enabled where available
* Failed login attempts must be monitored
* Compromised passwords must be changed immediately

## 4. Roles and Responsibilities

| Role                 | Responsibilities                                  |
| -------------------- | ------------------------------------------------- |
| Executive Management | Approve policy and support compliance             |
| IT Security Team     | Enforce password controls and monitor compliance  |
| Department Managers  | Ensure team compliance                            |
| All Employees        | Follow password requirements and report incidents |

## 5. Compliance

### 5.1 Monitoring

Password compliance will be monitored by the IT Security Team.

### 5.2 Reporting

Violations must be reported to the IT Security Team.

### 5.3 Auditing

Password controls will be reviewed annually.

## 6. Enforcement

### 6.1 Violations

Violations may result in:

* Verbal warning
* Written warning
* Suspension of access privileges
* Disciplinary action
* Termination of employment

### 6.2 Reporting Violations

Report violations to: [security@company.com](mailto:security@company.com)

## 7. Exceptions

### 7.1 Exception Process

Exceptions require:

1. Written request
2. Business justification
3. Risk assessment
4. Management approval

### 7.2 Exception Duration

Exceptions must be reviewed annually.

## 8. Definitions

| ------------ | ------------------------------------------------- |
| Password     | Secret string used for authentication             |
| MFA          | Multi-Factor Authentication                       |
| User Account | Authorized account used to access company systems |
