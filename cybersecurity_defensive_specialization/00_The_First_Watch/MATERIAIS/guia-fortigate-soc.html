1

O que é FortiGate?

FortiGate é um firewall empresarial da Fortinet. Ele pode existir como equipamento físico, máquina virtual ou serviço em nuvem. Normalmente fica entre a Internet e a rede interna da empresa.

Internet │ ▼ ┌───────────────┐ │ FortiGate │ │ Firewall / VPN│ │ IPS / NAT │ └───────┬───────┘ │ ┌─────┴────────────┐ │ │ Rede interna DMZ Computadores Servidores
Firewall

Controla conexões

Decide qual tráfego será permitido ou bloqueado conforme origem, destino, porta e regra.

Segurança

Detecta ameaças

Pode usar IPS, antivírus, filtro web, controle de aplicações e inspeção SSL.

Empresas

Protege redes corporativas

É comum em empresas, hospitais, bancos, órgãos públicos, universidades e data centers.

2

Como ler uma política de firewall

Leia sempre nesta ordem: origem, destino, serviço, ação, logs, status e NAT.

Exemplo FortiGate CLIconfig firewall policy
edit 1
    set name "Allow-Web-Inbound"
    set srcintf "wan1"
    set dstintf "dmz"
    set srcaddr "all"
    set dstaddr "web-srv-01"
    set action accept
    set service "HTTP" "HTTPS"
    set logtraffic all
    set status enable
next
CampoSignificadoPergunta para o analista
set nameNome da regraQual é o objetivo da regra?
set srcintfInterface de origemDe qual rede vem o tráfego?
set dstintfInterface de destinoPara qual rede ele vai?
set srcaddrIP ou grupo de origemQuem está iniciando a conexão?
set dstaddrIP ou grupo de destinoQual ativo está sendo acessado?
set serviceServiço ou portaQual protocolo está sendo usado?
set actionPermitir ou negarA conexão foi aceita ou bloqueada?
set logtrafficRegistro de logsExiste evidência para investigação?
set statusRegra ativa ou desativadaA regra está funcionando?
As regras são processadas de cima para baixo. Quando o tráfego combina com uma regra, o FortiGate aplica essa regra e normalmente interrompe a procura.
3

Interfaces mais comuns

WAN

Interface conectada à Internet. Exemplos: wan1, wan2.

Internal / LAN

Rede interna da empresa onde ficam computadores, impressoras e servidores internos.

DMZ

Zona separada para servidores públicos, como servidor web e servidor de e-mail.

VPN

Interface lógica usada por usuários remotos ou túneis entre filiais.

Guest

Rede de visitantes, normalmente isolada da rede corporativa.

Management

Interface dedicada à administração do equipamento.

4

Objetos de endereço

Objetos são nomes usados para representar IPs, redes ou grupos. Eles tornam as regras mais fáceis de entender.

Exemplo

  • web-srv-01 → 10.10.20.15
  • server-subnet → 10.10.20.0/24
  • finance-users → grupo de máquinas do financeiro

Por que isso importa?

Em uma investigação, o nome do objeto pode indicar rapidamente qual servidor, departamento ou rede está envolvido.

6

NAT

NAT traduz endereços IP. É muito usado para permitir que dispositivos com IP privado acessem a Internet.

Computador interno 192.168.1.50 │ ▼ FortiGate com NAT │ ▼ IP público da empresa 203.0.113.10 │ ▼ Internet
Em uma investigação, lembre-se: o IP público visto fora da empresa pode representar vários dispositivos internos. Para identificar o verdadeiro host, é necessário consultar os logs de NAT.
7

VPN

SSL VPN

Usada por funcionários remotos para acessar a rede corporativa.

IPsec VPN

Usada para criar túneis seguros entre filiais, data centers ou redes em nuvem.

O que verificar em logs de VPN

  • Nome do usuário
  • IP de origem
  • Horário da conexão
  • País ou localização aproximada
  • Falhas repetidas de autenticação
  • Acesso fora do horário normal
8

Logs e eventos

Os logs mostram o que o firewall permitiu, bloqueou ou detectou. Eles podem ser enviados para FortiAnalyzer, Wazuh, Splunk, Elastic, Sentinel ou outro SIEM.

Traffic log

Origem, destino, porta, ação, bytes e regra aplicada.

VPN log

Login, logout, falha de autenticação e criação de túnel.

IPS log

Exploração detectada, assinatura, severidade e destino.

Web Filter

Sites acessados, categorias, bloqueios e usuário.

Antivirus

Arquivo malicioso, nome da ameaça e ação tomada.

System log

Alterações de configuração, login administrativo e falhas do sistema.

9

Checklist para SOC Analyst

  • Identificar IP de origem e destino
  • Verificar porta, protocolo e serviço
  • Confirmar se a ação foi accept ou deny
  • Identificar a regra que autorizou ou bloqueou a conexão
  • Verificar se existe NAT e localizar o IP interno verdadeiro
  • Checar usuário e contexto de VPN
  • Procurar repetição, volume anormal ou padrão de ataque
  • Comparar o evento com IPS, antivírus e web filter
  • Verificar horário, localização e ativo afetado
  • Documentar evidências e recomendar contenção
