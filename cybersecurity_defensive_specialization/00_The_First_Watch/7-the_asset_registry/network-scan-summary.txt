SCAN ANOMALIES AND NOTES
========================================================================

1. UNDOCUMENTED DEVICES:
   - 10.10.2.99 (Central servers subnet) -- unknown Linux device with
     web services. Not in any IT documentation.
   - 10.10.10.200 (Westside) -- unknown Linux device with port 3000.
     Not in any IT documentation.

2. FLAT NETWORK CONFIRMATION:
   During scanning, all subnets were reachable from the scan host
   (Sarah's workstation at HQ, 10.10.20.10) without any access
   restrictions. This confirms Marcus's observation: no network
   segmentation is enforced. A device on any subnet can reach any
   other device on any other subnet.

3. END-OF-LIFE SYSTEMS:
   - 10.10.1.70 (WS-RAD-01): Windows XP SP3 -- the MRI workstation
   - 10.10.2.31 (print-srv-01): Windows Server 2012 R2 -- EOL Oct 2023
   - 10.10.2.15 (billing-srv-01): Ubuntu 18.04 -- standard support
     ended June 2023, ESM available but not activated

4. EXPOSED SERVICES:
   - MySQL on billing-srv-01 (10.10.2.15:3306) is accessible from the
     entire internal network. Should be restricted to ehr-srv-01.
   - PostgreSQL on ehr-db-01 (10.10.2.11:5432) is accessible from the
     entire internal network. Should be restricted to ehr-srv-01.
   - RDP (3389) is enabled on reception and admin workstations. No
     network-level restriction.
   - The NAS management interface (10.10.2.41:5000/5001) is accessible
     from the entire network.

5. MEDICAL DEVICE EXPOSURE:
   All Philips monitors, BD Alaris pumps and other medical devices
   have HTTP/HTTPS management interfaces accessible from the entire
   network. The BD Alaris firmware version (12.1.2) has known CVEs
   (BD issued a security bulletin 18 months ago recommending network
   isolation as mitigation -- this has not been done).
