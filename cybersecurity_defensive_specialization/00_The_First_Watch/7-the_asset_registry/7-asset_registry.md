## Reconciliation Notes

### Assets in the network scan but NOT in any documentation (Shadow IT / undocumented)

- **A-012 (UNKNOWN-01, 10.10.2.99):** unidentified Linux host with two web services, sitting on the same subnet as core servers. Sarah has no record of it; suspects Marcus or the IT intern.
- **A-025 (10.10.10.200, Westside):** unidentified Linux host with port 3000 open (commonly Grafana or Node.js). No documentation anywhere. Possibly an unofficial monitoring tool someone set up.
- **WS-WC-XRAY (10.10.10.100, Westside):** a vendor-specific X-ray workstation appears in the scan, but no prior source itemized it individually Task 0 only described Westside's imaging capability generically ("X-ray, ultrasound").

### Assets in documentation but NOT in the network scan

- **Second Westside server:** Marcus's note in Task 0 said "there might be another server in the closet" at Westside. It does not appear anywhere in the scan, which suggests it either does not exist or was powered off during the scan window.
- **A-019 (Pharmacy Management System):** named explicitly in the Task 1 incident log, but no host in the scan is clearly identified as running it it may be on an undocumented device, a vendor-hosted cloud service, or bundled into a system not separately fingerprinted.
- **Guest WiFi network (Central):** Task 0/3 both mention a separate guest SSID, but no distinct guest subnet appears anywhere in the scan results  raising the question of whether guest traffic is actually segmented at all, or just logically labeled.

### Discrepancies or contradictions between sources

- Task 2's diagnostics pointed to Apache 2.4.29 on `billing-srv-01` (A-004) as the likely entry point for both compromises; the network scan only shows port 80 open and does not independently confirm the web server version. This claim still needs direct verification, not just repetition.
- The medical device counts in Task 0 (~80 monitors, ~120 pumps) are roughly consistent with the scan once the "additional devices omitted for brevity" notes are factored in one of the few places where documentation and the independent scan actually agree.
- Task 0 described Westside as having its own "server closet for basic needs," implying a small, self-contained setup; the scan confirms only one server (A-024) there, alongside a full workstation fleet reaching the building over an unfirewalled consumer router the network exposure is more significant than the original "basic needs" description suggested.
